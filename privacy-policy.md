# Privacy Policy

Last updated: May 9, 2026

This Privacy Policy describes how Marketing Agent collects, uses, stores, and deletes data when you use the application or connect Meta assets such as Facebook Pages and Instagram professional accounts.

Marketing Agent is currently a private/beta tool. It is not offered as a public consumer service.

## Data we collect

Marketing Agent may collect or process the following data when you connect accounts or use the application:

- Account connection data, including Meta app-scoped user IDs, Facebook Page IDs, Instagram professional account IDs, usernames, Page names, and account metadata returned by Meta APIs.
- OAuth credentials, including access tokens needed to operate connected Facebook and Instagram assets.
- Content data, including captions, media references, drafts, scheduled posts, publishing results, comments, replies, and related campaign metadata that you create or authorize the app to manage.
- Operational data, including logs, error messages, token status, API response metadata, audit records, and security events.
- Workspace data that you enter into the app, such as brands, campaigns, contacts, notes, approvals, and marketing plans.

## How we use data

Marketing Agent uses data to:

- Authenticate connected Meta accounts and verify permitted access.
- Discover connected Facebook Pages and Instagram professional accounts.
- Create, schedule, publish, and monitor marketing content when requested by an authorized user.
- Maintain audit trails, prevent unauthorized activity, diagnose errors, and improve reliability.
- Comply with platform requirements, legal obligations, and user deletion requests.

## Sharing and sale of data

Marketing Agent does not sell personal data.

Data is shared only where needed to operate the application, such as sending authorized requests to Meta APIs, using infrastructure or service providers selected by the app operator, complying with law, or protecting the security and integrity of the service.

## Storage and security

Marketing Agent is designed for controlled private use. Access tokens and credentials should be stored in secure environment variables, operating-system keychains, or other secret-management systems. The app operator is responsible for securing the environment where Marketing Agent runs.

No system can guarantee absolute security. If a credential is exposed, revoke it in the relevant provider dashboard and rotate it immediately.

## Retention

Marketing Agent keeps data only as long as needed for the purposes described in this policy, unless a longer retention period is required for legal, security, backup, or audit reasons. OAuth tokens and connection records are deleted when a connected account is removed or a verified deletion request is processed.

Backups and logs may retain limited data for a short period after deletion. Where immediate deletion is not practical, data is removed or anonymized during the normal backup and log-retention cycle.

## User choices and deletion

You can revoke Marketing Agent's access to Meta data from Facebook's Apps and Websites settings. You can also request deletion of data held by Marketing Agent.

See the Data Deletion Instructions:

https://gzaal.github.io/marketing-agent-policies/data-deletion/

## Children's privacy

Marketing Agent is not directed to children and is not intended for use by anyone under 16 years old.

## International use

If you use Marketing Agent from outside the country where the app operator runs the service, your data may be processed in that operator's jurisdiction or in locations where selected service providers operate.

## Changes

This policy may be updated as Marketing Agent changes. The "Last updated" date above reflects the latest version.

## Contact

For privacy questions or deletion requests, use the contact method listed in the Meta App Dashboard or open a request at:

https://github.com/gzaal/marketing-agent-policies/issues

Do not post access tokens, passwords, private credentials, or sensitive personal data in a public GitHub issue. Ask for a private contact channel first if the request requires private details.
